# 글쓰기 레퍼런스 수집과 AI티 검수 원칙

작성 시각: 2026-06-28T13:20+09:00

## 목적

이 프로젝트의 note 글, Vercel 콘텐츠, 커뮤니티 게시문, 오프라인 안내문은 단순히 정보를 전달하는 글이 아니다. 타겟이 실제로 쓰는 말투, 감정의 순서, 고민의 구조 안으로 들어가야 검증 응답이 나온다.

따라서 새 글을 쓸 때는 먼저 유사한 일본어 글을 수집하고, 그 글들의 형태와 감정 흐름을 참고한 뒤, 기존에 우리가 쓴 글이 AI처럼 보이지 않는지 적대적으로 검수한다.

## 적용 대상

- note 게시글
- Vercel 콘텐츠형 MVP 글
- X/Threads/Konest/커뮤니티 게시문 초안
- 오프라인 플라이어와 QR 안내문
- 제품 설명처럼 보일 수 있는 모든 소비자-facing 문안

## 작성 전 레퍼런스 수집

글을 새로 쓰거나 크게 고칠 때는 가능한 한 같은 채널/같은 독자/같은 상황의 일본어 글을 먼저 모은다.

- 최소 3개, 가능하면 5~7개의 유사 글을 본다.
- 레퍼런스는 실제 URL, 캡처 위치, 검색어, 접근 시각을 남긴다.
- 참고 대상은 판매 카피만이 아니라 후기, 일상 에세이, 고민 상담, 음식 블로그, note 글, SNS 짧은 글까지 포함한다.
- 단순 주제 유사성보다 `누가 어떤 마음으로 읽는가`가 더 중요하다. 예를 들어 참기름 글이면 `韓国料理`, `新大久保`, `ごま油`, `使い切れない`, `香り`, `韓国スーパー`, `冷奴`, `卵かけご飯` 같은 실제 맥락의 글을 우선한다.

레퍼런스를 볼 때는 문장을 베끼지 말고 아래를 분리해 기록한다.

| 관찰 항목 | 볼 것 |
|---|---|
| 글 형태 | 일기형, 후기형, 질문형, 정보 정리형, 가벼운 모집형, 체험담형 중 무엇인지 |
| 감정의 흐름 | 기억/상황 → 작은 불편 → 비교/망설임 → 질문/행동 요청 순서가 어떻게 움직이는지 |
| 문체 | 딱딱한 설명인지, 혼잣말인지, 친구에게 묻는 말인지, 판매자 말투인지 |
| 구조 | 제목, 첫 문장, 중간 전환, 리스트 사용 여부, CTA 위치 |
| 구체성 | 매장명, 가격, 용량, 요리명, 계절, 구매 장면 같은 세부가 얼마나 들어가는지 |
| 상업성 | 판매 느낌을 숨기는지, 솔직히 밝히는지, 어느 지점에서 거부감이 생기는지 |

## 초안 작성 기준

- 레퍼런스의 고유 문장, 제목, 비유, 독특한 표현은 베끼지 않는다.
- 참고하는 것은 `형태`, `감정의 순서`, `구체성의 밀도`, `CTA의 거리감`이다.
- 검증 글은 제품 장점을 먼저 말하지 말고, 독자가 이미 겪었을 법한 장면과 망설임을 먼저 꺼낸다.
- `한국식이라 좋다`, `향이 진하다`, `작아서 좋다` 같은 일반 문구는 약하다. 반드시 현재 대체재와 비교되는 장면을 넣는다.
- CTA는 결제/예약/개인정보 수집으로 보이지 않게 하고, “기억나는 범위에서 알려달라”는 낮은 마찰로 둔다.

## AI티 적대적 검수 체크리스트

게시 전 또는 문안 수정 전에는 기존에 쓴 글과 새 초안을 다시 읽고 아래를 확인한다.

- 너무 반듯한 문제-해결-CTA 구조라 사람 글보다 기획서처럼 보이지 않는가.
- 제목이 일반 광고 카피처럼 보이지 않는가.
- 첫 문장이 독자의 실제 상황이 아니라 제품 설명으로 시작하지 않는가.
- 리스트가 많아져서 감정 흐름이 끊기지 않는가.
- `需要確認`, `正式販売前`, `検証` 같은 말이 너무 반복되어 차갑게 보이지 않는가.
- 요리명, 매장명, 용량/가격, 사용 기간 같은 손에 잡히는 디테일이 부족하지 않은가.
- 독자가 거절할 수 있는 선택지, 예를 들어 `5gパックで十分`, `110ml小瓶で十分`, `新大久保で十分`이 열려 있는가.
- 답변자가 뭘 적어야 할지 바로 떠오르는가.
- 글이 너무 한국어 사고방식을 일본어로 번역한 느낌이 아닌가.

문제가 보이면 게시하지 말고 수정한다. 수정은 `더 감성적으로`가 아니라 `더 실제 장면에 가깝게`, `더 덜 판매자처럼`, `더 구체적인 반례를 허용하게` 고친다.

## 기록 방식

새 글을 만들거나 크게 고칠 때는 원고 패킷이나 KB에 아래를 남긴다.

```text
레퍼런스:
- URL / 글 제목 / 확인 시각 / 왜 참고했는지

참고한 것:
- 글 형태
- 감정 흐름
- 문체와 CTA 거리감

베끼지 않기로 한 것:
- 특정 제목/문장/표현
- 과하게 상업적인 구조

AI티 검수:
- 발견한 어색함
- 수정한 내용
- 그래도 남은 리스크
```

## 운영 규칙

- 레퍼런스를 수집하지 못했으면 `레퍼런스 없음`을 명시하고, live 게시 전에는 가능하면 다시 수집한다.
- 이미 게시한 글도 필요하면 다시 읽고 AI티를 검수한다. 문제가 크면 새 게시물을 늘리기보다 기존 글/CTA/문장 구조를 먼저 고친다.
- 레퍼런스 수집 결과와 수정 이유는 `검증/게시_배포_기록.md`, 해당 `mvp/*_posting_packet.md`, 또는 별도 KB 문서에 남긴다.
- 레퍼런스는 출처로 남기되, 문장/구조를 과도하게 모방하지 않는다. 목표는 표절이 아니라 타겟 언어 감각과 감정 흐름을 맞추는 것이다.
